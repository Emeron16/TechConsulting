# Requesting Elevated / Administrator Access

By default, TechConsulting employees do not have local administrator rights on company-issued devices. This is a security policy, not a technical limitation, and it cannot be granted through self-service, automated tooling, or the IT chatbot/assistant under any circumstances.

## Why standard accounts don't have admin rights

Standard (non-admin) accounts significantly reduce the impact of malware, ransomware, and phishing attacks, since most attacks require administrator privileges to make persistent changes to a device. This is a company-wide security control, not a per-employee restriction.

## How to request temporary elevated access

1. Submit a request through the **TC Helpdesk Portal** (helpdesk.techconsulting.com), category "Elevated Access Request."
2. Include a specific business justification (for example, "installing a licensed development tool not in the App Store that requires an admin installer").
3. Your request is routed to your manager for approval, then to IT Security for a final review.
4. If approved, IT Security grants **time-limited** elevated access (typically 4–8 hours) via Privileged Identity Management (PIM), which automatically expires and reverts your account to standard access without any action needed on your part.

## What is never granted

Permanent local administrator rights, the ability to disable antivirus/endpoint protection, and the ability to install unapproved software outside the TC App Store are not granted under any request, regardless of justification or approval level.

## Automated assistant note

If you're asking the IT assistant to perform an action that requires administrator rights (registry changes, installing unapproved software, disabling security agents, creating local user accounts, etc.), it will not attempt this on your behalf. Use the request process above instead.
